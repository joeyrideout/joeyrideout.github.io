Configuring Octopress on GitHub Pages Using Windows 8

Jul 7th, 2013

I recently finished getting this Octopress blog running on GitHub Pages, and I couldn’t be happier with the result. It took a bit of work to get up and running, however, so I figured that I should document some of the hurdles I had to jump through on Windows to get this blog deployed. The default installation instructions for Octopress were generally okay, but a couple of unexpected errors arose that left me hanging.

Of course, you need to install Git in order to start working with GitHub Pages. However, Octopress did not cooperate with my Git installation until I added Git.exe to my system’s PATH. To do that, right-click on My Computer, select Properties, and then Advanced system settings. From the Advanced tab that you are currently on, click the Environment Variables button at the bottom. Edit Path and add the directory with your newly installed Git executable, separated from the last entry with a semicolon. For example, …;C:\Program Files (x86)\Git\bin

You will also need to install Ruby 1.9.3 as well as the Ruby DevKit that Octopress depends on. The original Octopress guide does not provide Windows installation instructions for Ruby, but you can find some convenient Windows installers here.

After getting those software packages installed, you should be able to follow the rest of Octopress’ installation instructions without any problems. I will paraphrase the steps below (which I executed in Windows Powershell).

Now that Octopress is installed, it’s time to deploy your site to GitHub. Note that all of your blog administration will be run from the /GitHub/octopress directory, and not from your site’s directory that will be generated in the next step.

Assuming you have already signed up with GitHub, create a repository username.github.io where username is your GitHub username.
